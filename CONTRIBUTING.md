# Contributing to git-gud

First off — thanks for being here. Whether you're following this roadmap yourself or want to improve it for others, you're welcome.

## How to Use This Roadmap

1. Fork this repo
2. Follow the tracks at your own pace
3. Track your progress in the templates (`docs/templates/`)
4. Build the projects yourself — no copy-pasting code

## How to Suggest Improvements

Found a broken link? Better resource? Typo? Open an issue or PR.

### What makes a good contribution:
- Fix broken links or outdated resources
- Suggest better learning resources (with explanation of why)
- Add project ideas that reinforce concepts
- Fix typos or improve clarity
- Share your own progress (inspires others)

### What we don't accept:
- AI-generated content dumps
- Low-effort "awesome list" style additions without context
- Changes that contradict the core philosophy (Foundation Before Framework)
- Removing the "no AI for coding" stance — that's intentional

## Code of Conduct

Be kind. Everyone here is learning. Nobody was born knowing JavaScript.

## Questions?

Open an issue. There's no such thing as a dumb question — only dumb answers from people who forgot what it was like to be a beginner.
