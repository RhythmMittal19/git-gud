🏗️ 07 - Semantic HTML

Semantic HTML uses elements that describe their meaning, not just their appearance.

🤔 Why Use Semantic HTML?

Benefits
Benefit Explanation
♿ Accessibility Screen readers understand page structure
🔍 SEO Search engines rank content better
👨‍💻 Maintainability Easier for developers to read and update
📱 Consistency Browsers apply default styling

❌ Div Soup vs ✅ Semantic HTML

❌ DIV SOUP (Bad)
<div class="header">
<div class="nav">...</div>
</div>
<div class="main">
<div class="article">...</div>
<div class="sidebar">...</div>
</div>
<div class="footer">...</div>
✅ SEMANTIC (Good)
<header>
<nav>...</nav>
</header>
<main>
<article>...</article>
<aside>...</aside>
</main>
<footer>...</footer>
💡 Both look the same visually! But semantic HTML gives MEANING to the structure. Screen readers and search engines understand it better.

📦 Semantic Elements Reference

Element Purpose Use Case
<header> Introductory content Page header, article header
<nav> Navigation links Main menu, breadcrumbs
<main> Main content Primary page content (ONE per page)
<article> Self-contained content Blog post, news article, comment
<section> Thematic grouping Chapters, tabbed content
<aside> Related but separate Sidebar, pull quotes, ads
<footer> Footer content Copyright, links, contact info
<figure> Self-contained media Images, diagrams, code snippets
<figcaption> Caption for figure Image description
<time> Date/time Published date, event time
<mark> Highlighted text Search results highlight
<details> Expandable content FAQs, collapsible sections
<summary> Summary for details Click to expand label

📂 Expandable Content (details/summary)

CODE

</details>

💡 No JavaScript needed! The <details> element creates expandable/collapsible content using pure HTML.

🕐 Time Element

<!-- Human readable + Machine readable -->
<p>
Published on <time datetime="2026-01-15">January 15, 2026</time>
</p>
<p>
Event starts at <time datetime="2026-01-15T19:30">7:30PM</time>
</p>
<!-- The datetime attribute is for machines(search engines, calendars) -->
<!-- The text content is for humans-->

✅ HTML Complete Checklist

Before Moving to CSS

Make sure you can do all of these:

  • ☐ Write HTML document structure from memory
  • ☐ Use all text elements (h1-h6, p, strong, em, etc.)
  • ☐ Create links (external, internal, anchor, email)
  • ☐ Add images with proper alt text
  • ☐ Create all types of lists (ul, ol, dl)
  • ☐ Build tables with thead, tbody, colspan, rowspan
  • ☐ Create forms with various input types
  • ☐ Use semantic HTML elements correctly
  • ☐ Validate your HTML (no errors)
