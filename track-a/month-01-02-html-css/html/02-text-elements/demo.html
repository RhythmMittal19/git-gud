📝 02 - Text Elements

HTML provides various elements to structure and format text content.

📌 Headings (h1 - h6)

HTML has 6 levels of headings. <h1> is the most important, <h6> is the least.

CODE
<h1>Heading 1</h1>
<h2>Heading 2</h2>
<h3>Heading 3</h3>
<h4>Heading 4</h4>
<h5>Heading 5</h5>
<h6>Heading 6</h6>
OUTPUT

Heading 1

Heading 2

Heading 3

Heading 4

Heading 5
Heading 6
⚠️ Important Rules:
  • Only ONE <h1> per page (like a book title)
  • Don't skip levels (h1 → h3 is wrong, use h1 → h2 → h3)
  • Use headings for STRUCTURE, not for making text big

📌 Paragraphs

CODE
<p>This is a paragraph. Browsers add space above and below automatically.</p>
<p>This is another paragraph.</p>
OUTPUT

This is a paragraph. Browsers add space above and below automatically.

This is another paragraph.

💡 Note: Line breaks in your code are ignored. The text flows continuously. Use <br> for line breaks.

📌 Line Break & Horizontal Rule

CODE
<p>Line one<br>Line two<br>Line three</p> <hr> <p>Content after the line.</p>
OUTPUT

Line one
Line two
Line three

Content after the line.

📌 Text Formatting

Element Code Output Use When
Strong (Bold) <strong>text</strong> text Important content
Emphasis (Italic) <em>text</em> text Stressed words
Bold (visual only) <b>text</b> text Stylistically bold (no importance)
Italic (visual only) <i>text</i> text Technical terms, foreign words
Underline <u>text</u> text Misspelled words, proper names (Chinese)
Strikethrough <del>text</del> text Deleted content
Inserted <ins>text</ins> text Added content
Highlight <mark>text</mark> text Highlighted/marked content
Small <small>text</small> text Fine print, copyright
Subscript H<sub>2</sub>O H2O Chemical formulas
Superscript x<sup>2</sup> x2 Math exponents, footnotes
Code <code>text</code> text Inline code

📌 <strong> vs <b> — What's the Difference?

They LOOK the same, but MEAN different things:
<strong> = IMPORTANT

Warning: Do not touch!

Screen readers will emphasize this.
It has semantic meaning.

<b> = Just bold styling

Product Name

Just visual styling.
No special meaning.

💡 Rule of Thumb: If the text is IMPORTANT, use <strong>. If you just want it bold for styling, use CSS instead.

📌 Preformatted Text

The <pre> tag preserves spaces and line breaks:

CODE
<pre> This preserves all spaces and line breaks </pre>
OUTPUT
    This    preserves
    all     spaces
    and line breaks

📌 Blockquote

CODE
<blockquote> "The only way to learn to code is to code." </blockquote>
OUTPUT
"The only way to learn to code is to code."
