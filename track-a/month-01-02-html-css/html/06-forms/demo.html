📝 06 - Forms

Forms collect user input. They're essential for login pages, contact forms, surveys, and more.

📋 Basic Form Structure

<form action="/submit" method="POST">
<!-- Form elements go here -->
</form>
action = Where to send data (URL)
method = How to send (GET or POST)
💡 GET vs POST:
  • GET: Data visible in URL (use for searches)
  • POST: Data hidden (use for login, sensitive data)

⌨️ Text Input Types

Text Input

CODE
<label for="name">Name:</label>
<input type="text" id="name" name="name" placeholder="Enter your name" required >
OUTPUT

Email Input

CODE
<label for="email">Email:</label>
<input type="email" id="email" name="email" placeholder="you@example.com" required >
OUTPUT (validates email format!)

Password Input

CODE
<label for="password">Password:</label>
<input type="password" id="password" name="password" minlength="8" required >
OUTPUT (characters hidden!)

🏷️ Important Form Attributes

Attribute Purpose Example
required Field must be filled <input required>
placeholder Hint text inside input placeholder="Enter name"
disabled Can't interact with field <input disabled>
readonly Can read but not edit <input readonly>
minlength Minimum characters minlength="8"
maxlength Maximum characters maxlength="100"
min / max Number range min="0" max="100"
pattern Regex validation pattern="[A-Za-z]+"
autofocus Focus on page load <input autofocus>
autocomplete Browser autocomplete autocomplete="email"

🏷️ Why Labels Matter

⚠️ Always use labels with for attribute!
  • Clicking label focuses the input (better UX)
  • Screen readers read labels (accessibility)
  • The for must match input's id
<!-- ❌ Wrong -->
Name:<input type="text">
<!-- ✅ Correct -->
<label for="name">Name:</label>
<input type="text" id="name">
